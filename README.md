
这是一个为 [AstrBot](https://github.com/Soulter/AstrBot) 框架深度定制的BF1群管辅助插件。主要用于配合游戏服务器的“GHS管服机器人”，自动监听并记录玩家被踢出的违规历史。
## ✨ 核心特性

- 🤖 **自动监听**：精准捕捉GHS管服机器人发送的踢人播报，自动记录玩家 ID、违规原因及精确时间。
- 🛡️ **安全校验**：
  - **来源白名单**：只监听指定“管服机器人”发送的消息，防止伪造记录。
  - **管理权限池**：只有指定的超级管理员（或群主/群管理员）才能执行清空数据的敏感操作。
- ⚙️ **WebUI 原生配置**：适配 AstrBot 官方设置面板，动态增删管理员和机器人名单，即改即生效。
- 💾 **无损数据持久化**：数据存储于本地 JSON 文件，自动兼容老版本数据格式，更新重启不丢数据。

## 📦 安装指南

1. 确保你已安装并运行 [AstrBot](https://github.com/Soulter/AstrBot) 框架。
2. 在 AstrBot 的 `data/plugins/` 目录下创建一个新文件夹，命名为 `astrbot_plugin_tk_tracker`（或你在 metadata 中定义的名称）。
3. 将本项目的 `main.py` 和 `_conf_schema.json`（以及 `metadata.yaml`） 放入该文件夹中。
4. 打开 AstrBot 控制台，点击 **“重载插件”**。

## ⚙️ 配置说明 (WebUI)

1. 登录 AstrBot 的 WebUI 管理后台。
2. 在“插件管理”列表中找到本插件。
3. 在弹出的可视化面板中，填写以下信息：
   - **超级管理员 QQ 号**：拥有清空违规记录最高权限的人。
   - **管服机器人 QQ 号**：负责在群里发送踢人播报的机器人账号。
4. 点击保存后，再次点击 **“重载插件”** 即可立即生效

## 🚀 指令清单

| 指令 | 参数 | 描述 | 权限要求 |
| :--- | :--- | :--- | :--- |
| `/tk帮助` | 无 | 获取插件的使用说明书。 | 所有人可用 |
| `/tk查` | `[玩家ID]` | 查询指定玩家的累计被踢出次数及最近 5 次的详细违规历史。 | 所有人可用 |
| `/tk排行` | 无 | 显示kick排行榜 | 所有人可用 |
| `/tk清空` | `[玩家ID]` | 清除指定玩家的所有违规记录。 | 仅限管理员 |
| `/tk清空全部` | 无 | ⚠️ 危险操作：清空数据库内所有人的违规记录 | 仅限管理员 |
## 📝 触发格式要求

插件会自动识别包含以下三个关键词的群聊消息：
- `踢出玩家` (其后紧跟玩家ID)
- `成功`
- `原因` (其后紧跟具体违规理由)

*只要消息中同时包含以上内容（无论中间是否有换行或空格），且发送者为配置面板中登记的管服机器人，即会自动记录。*

## 📁 数据存储位置

插件的运行数据默认保存在 AstrBot 根目录下的：
`data/tk_tracker/records.json`
